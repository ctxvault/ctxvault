# CtxVault v0.4.0 Release Notes

Status: public Day 0 release for the local trust-and-handoff package.

## Positioning

v0.4.0 packages CtxVault as a complete local trust-and-handoff product
candidate:

- Core deterministic source-to-context-to-projection flow
- Demo Vault path with no account, no cloud, and no LLM key
- static Receipt/Trust Gallery
- artifact-only feedback path
- optional notarized Mac App Alpha artifact distributed through the GitHub
  release assets

Primary hook:

`Know what your AI tools see.`

Category line:

`The local trust layer for AI work.`

## Added

- static `release/v0.4.0/receipt-trust-gallery/` examples generated from
  sanitized deterministic demo evidence
- release notes for the v0.4.0 local trust-and-handoff package
- package metadata updated to v0.4.0
- release asset path for an optional notarized Mac App Alpha artifact

## Preserved

- no provider call in the default demo path
- no local or remote LLM required for baseline correctness
- no hidden session scanning
- no runtime control over Codex, Claude Code, Cursor, terminals, browsers, or
  hosted tools
- no automatic durable memory promotion
- review-gated projection and receipt-backed writes

## Known Limits

- Product Hunt is not part of the first batch.
- Show HN is Day 1 only after Day 0 artifacts are live and immediately usable.
- Public Workbench beta, external session discovery, provider/model execution,
  telemetry, cloud sync, and runtime control remain out of scope.
