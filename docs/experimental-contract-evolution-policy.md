# CtxVault Experimental Contract Evolution Policy

These public contracts are real but explicitly experimental:

- `intelligence.py`
- `Episode`
- `Workstream`
- compiled workstream state read model
- doctor report
- plugin manifest contract
- projection receipt contract
- the first local plugin executor path

Experimental means:

- they are inspectable and intentionally shipped
- they may still change before becoming long-term compatibility promises
- changes should be reflected in docs, fixtures, and tests together
