# CtxVault v0.3 Compiled Context

v0.3 is the compiled Context Injection milestone for the public `ctxvault`
core.

It keeps CtxVault's canonical truth in governed local objects, then builds
rebuildable read models and projections for AI work surfaces.

## What v0.3 Adds

- experimental compiled workstream state read model
- compiled current truth in `AGENTS.md`, `CLAUDE.md`, and workstream brief
  projections
- source refs and advisory warnings in generated projections
- read-only `doctor` diagnostics
- Markdown-vault import bridge for `.md` and `.markdown` notes
- CLI and MCP access to compiled state and doctor

## Compiled Workstream State

Compiled state is a read model. It is derived from reviewed `Workstream`,
`Memory`, `KnowledgeArtifact`, receipts, and source refs.

It is not a new canonical object and it is not stored as durable truth.

The public schema id is:

```text
ctxvault.compiled-workstream-state/v1
```

## Doctor

`ctxvault doctor` is read-only in v0.3. It reports local diagnostic state such
as vault/index presence, fixtures, launch-gate evidence, compiled-state
contract availability, and projection adapter healthchecks.

It does not auto-fix, contact remote services, call model endpoints, or mutate
workspace files.

## Markdown-Vault Bridge

`markdown-vault-import` imports Markdown files as governed source material.

The bridge does not make Obsidian or any Markdown vault the canonical store.
Durable truth remains in CtxVault objects and projection receipts.

## Non-Goals

- official Obsidian plugin
- model adapters in the baseline
- hybrid search or typed graph as public promises
- autonomous enrichment or promotion
- public workbench or signed wrapper availability
