# CtxVault v0.3.1 Release Notes

Status: v0.3.1 local safety and context slicing milestone.

## Positioning

v0.3.1 keeps the v0.3 compiled Context Injection path and adds a safer local
selection layer. The release is still model-free at the deterministic baseline:
context is sliced, searched, preflighted, and projected locally with receipts.

## Added

- experimental `ctxvault.context-slice/v1` schema and fixture
- deterministic context slicing for Markdown knowledge, session turns,
  episodes, and compiled workstream state
- `context-slice-rebuild` CLI command
- `context-search` CLI command
- `context-selection-preflight` CLI command
- MCP tools for context search and selected-slice privacy preflight
- privacy classes for searchable plain text, searchable redacted previews,
  metadata-only slices, and withheld slices
- selected-slice projection preflight before `AGENTS.md`, `CLAUDE.md`, and
  workstream brief writes
- projection receipts carrying selected slice refs and embedded privacy
  preflight evidence
- `ctxvault.privacy-preflight-receipt/v1` schema and fixture
- review-gated logical purge planning and apply paths for derived slice rows,
  FTS rows, preview data, optional embedding rows, and selected generated
  projection files
- logical purge tombstones for projection receipts that referenced purged
  derived data
- `doctor` slice-index health, stale-source checks, and projection slice-ref
  diagnostics

## Preserved

- canonical truth stays in governed source objects
- context slices are rebuildable read models
- selected context is blocked before projection when privacy preflight returns
  `block`
- projection and purge evidence remains receipt-based
- Markdown-vault import remains a bridge, not canonical storage
- baseline correctness has no model, embedding, remote service, hosted API, or
  vector database dependency

## Explicit Non-Goals

- no automatic source-material clearing
- no SSD, APFS snapshot, backup, cache, or filesystem secure-wipe claim
- no vector search baseline
- no graph product claim
- no model adapter baseline
- no silent remote reranking, embedding, enrichment, or durable truth promotion
- no official Obsidian plugin
- no Codex, Claude Code, Cursor, shell, or AI-chat live connector claim
